# Kanji Roguelike Game Specifications

## 1. システム概要
漢字の合体を核としたデッキ構築型ローグライク。
プレイヤーは20枚の固定デッキを編成し、戦闘中に漢字を合体させながら敵を倒す。

## 2. デッキシステム
- **デッキサイズ**: 固定20枚。
- **カード制限**: 同名漢字は最大3枚まで。
- **管理**: `DeckManager`が実データを保持。インベントリからデッキを編成するUIがフィールド画面で利用可能。

## 3. 戦闘メカニクス
- **ドロー**: ターン開始時にデッキから5枚ドロー。
- **合体**: 手札のカードを重ねることで合体。
  - **AP回復**: 合体成功時にAP（マナ）が+1回復。
  - **スロー演出**: 合体成功時に一瞬（0.2倍速）スローになり、爽快感を強調。
- **AP消費**: 各カードの使用にマナを消費。

## 4. プロジェクト構成
- **最適化**: `.gitignore` および `.gitattributes` (LFS) 設定済み。
- **シングルトン**: `GameManager`, `BattleManager`, `VFXManager` が各機能を統括。
- **データ管理**: ScriptableObject (`KanjiCardData`, `KanjiFusionDatabase`) を使用。

## 5. 演出 (VFX)
- `VFXManager`によるスローモーション、シェイク、閃光エフェクト。
- 漢字の合体による進化演出。
